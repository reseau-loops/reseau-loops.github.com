Action de formation sur les tests

Le réseau national DevLOG organise une série d'actions sur les tests logiciels dans la continuité d'ENVOL 2014. DevelopR6 accueille et organise localement une de ces sessions en 2016.

Les tests, et plus généralement la validation des logiciels constituent une phase primordiale dans le processus de développement. Ils représentent l'un des principaux leviers de la réussite d'un projet de développement d'applications. Par une meilleure connaissance des méthodes et outils comme les tests unitaires, fonctionnels, de non-régression, d'intégration, ... les participants seront capables d'identifier les étapes successives de validation du code et de leur pilotage au travers du cycle de développement, avec par exemple le processus d'intégration continue.

Date, horaires

Du mercredi 16/11/2016 au vendredi 18/11/2016

Horaires : de 9h à 17h

Programme

16/11 : Test Unitaire et Structurel

  • 9h-12h : Le test structurel

    • Comprendre les concepts : méthodes et critères
    • Mise en œuvre pratique (sur l'exemple fil rouge de la formation)

  • 13h30-17h : Test unitaire

    • Savoir exécuter les tests
    • Comprendre la structure d'un test
    • Comprendre et utiliser la couverture pour qualifier les tests
    • Gérer le problème des méthodes privées

17/11 : Tests unitaire et d'acceptation

  • 9h-11h : Test unitaire 2ème partie

    • Savoir gérer l'évolution du code et des tests
    • Savoir tester en isolation
  • 11h-12h : Gérer des tests dans une démarche d'Intégration continue
  • 13h30-14h30 : Test fonctionnel - les concepts

  • 14h30-17h Test d'acceptation

    • Comprendre l'utilisation du test d'acceptation (échange avec le client, niveau d'utilisation)
    • Comprendre et utiliser un framework pour le test d'acceptation (concordion)

18/11 Test fonctionnel

  • 9h-11h : Test d'application web

    • Comprendre et utiliser un framework pour le test d'application web (selenium)
    • Intégrer les tests fonctionnels dans une démarche d'intégration continue

  • 11h-12h : Exigences

    • Comprendre l'utilisation des exigences dans une démarche de test
    • Appréhender l'écriture d'une exigence dans le but de la tester

  • 13h30-17h Des exigences au test d'application web

    • Comprendre l'utilisation du test fonctionnel (niveau d'utilisation, besoin et complémentarité avec TU)
    • Etablir la traçabilité entre exigences et tests de l'application
    • Automatisation des tests
    • Production de rapports
